# Markdown Viewer — Iteration 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a lightweight desktop Markdown viewer (no editing) that opens `.md` files and renders sanitized HTML.

**Architecture:** Tauri v2 desktop app. A vanilla HTML/CSS/JS webview frontend calls Rust IPC commands. `parse_markdown` (pulldown-cmark + ammonia) converts markdown to sanitized HTML; `read_markdown_file` reads a chosen file from disk. The frontend injects the HTML into a single viewport. The OS file dialog runs in JS so Rust only ever receives a path, keeping permissions minimal (dialog only).

**Tech Stack:** Rust, Tauri v2, pulldown-cmark, ammonia, Vite, vanilla JS, `@tauri-apps/plugin-dialog`.

```rust
#[tauri::command]
pub fn parse_markdown(content: String) -> Result<String, String> {
    let mut options = Options::empty();
    options.insert(Options::ENABLE_TABLES);
    options.insert(Options::ENABLE_STRIKETHROUGH);
    options.insert(Options::ENABLE_FOOTNOTES);
    options.insert(Options::ENABLE_TASKLISTS);

    let parser = Parser::new_ext(&content, options);
    let mut rendered = String::new();
    html::push_html(&mut rendered, parser);

    Ok(ammonia::clean(&rendered))
}
```

## Task 6: Wire the frontend IPC logic

**Files:**
- Replace: `src/main.js`

- [ ] **Step 1: Replace `src/main.js`**

Replace the entire contents of `src/main.js` with:

```js
import { invoke } from "@tauri-apps/api/core";
import { open } from "@tauri-apps/plugin-dialog";

const viewport = document.querySelector("#viewport");

const SAMPLE = `# Welcome to Groot

A lightweight **Markdown viewer** built with Tauri + Rust.

- Click **Open File** above to view a \`.md\` file.
- Rendering is powered by \`pulldown-cmark\`.
- Output is sanitized with \`ammonia\`.

## Example code

\`\`\`
fn main() {
    println!("hello, groot");
}
\`\`\`

> Editing is coming in a later iteration.
`;

function showError(message) {
  viewport.innerHTML = `<p class="error">⚠️ ${message}</p>`;
}

async function render(markdown) {
  try {
    viewport.innerHTML = await invoke("parse_markdown", { content: markdown });
  } catch (e) {
    showError(String(e));
  }
}

async function openFile() {
  try {
    const path = await open({
      multiple: false,
      filters: [{ name: "Markdown", extensions: ["md", "markdown"] }],
    });
    if (!path) return; // user cancelled
    const content = await invoke("read_markdown_file", { path });
    await render(content);
  } catch (e) {
    showError(String(e));
  }
}

window.addEventListener("DOMContentLoaded", () => {
  document.querySelector("#open-file").addEventListener("click", openFile);
  render(SAMPLE);
});
```
